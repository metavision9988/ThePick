## 🎩 메피스토의 개막 진단

지금까지 축적한 **9개 문서와 12개 기술 리서치**는 정교합니다. 구현 재정립서 v2.0, 구현 설계서 v2.0, 프론트엔드 마스터플랜, QA 마스터플랜 102개 테스트 — 이것들은 전부 **기술 설계서**로서 우수합니다.

그러나 Opus 4.7의 시선으로 냉정히 말하면:

> **"이 프로젝트는 '훌륭한 엔진'을 설계했다. 하지만 엔진이 실릴 '자동차'와 그 자동차가 달릴 '도로'는 아직 없다."**

**지금 Claude Code에 넘겨서 16주간 완벽하게 구현해도, 아무도 안 쓰는 서비스가 될 수 있습니다.** 왜냐하면 지금까지의 모든 논의가 **"어떻게 만들 것인가"**에만 집중되었고, **"왜 만드는가", "누가 살 것인가", "얼마에 팔 것인가", "법적으로 가능한가"**에 대한 답이 전무하기 때문입니다.

이것이 진산님이 느꼈던 "취미/장난감 vs 상용 서비스"의 차이의 **진짜 근원**입니다.

## 1️⃣ 시장 데이터가 드러낸 충격적 사실

첫 번째 단계로 손해평가사 시험의 **실제 시장 데이터**를 확보했습니다. 결과는 우리의 암묵적 전제를 흔듭니다.

### 실제 숫자

| 항목                  | 값                          | 출처        |
| --------------------- | --------------------------- | ----------- |
| 2024 2차 응시자       | 9,584명                     | 박문각/큐넷 |
| 2024 2차 합격자       | 566명                       | 박문각      |
| 2024 2차 합격률       | **5.9%** (2020년 이래 최저) | 공고        |
| 1차 합격률            | 60~70%                      | Q-Net 통계  |
| 주 합격 연령대        | **50대 (55.8%)**            | 박문각 2024 |
| 연간 총 수험생 모집단 | ~1만 명 (1차+2차)           | 추정        |

### 이 숫자가 뜻하는 것

**A. 시장이 생각보다 작다**

연간 응시자 1만 명. 10% 점유율을 달성해도 MAU 1,000명. 월 2만 원 구독이면 **매출 상한 월 2,000만 원, 연 2.4억**입니다.

이것이 사업인가? 개발자 1인의 생활비와 인프라 비용을 감당할 수 있는 수준이지만, **팀을 꾸리고 다른 시험으로 확장할 투자 자본을 만들기는 어렵습니다.**

**B. 주 고객이 50대다**

이것이 가장 중요한 발견입니다. 우리는 지금까지 **"모바일 퍼스트 PWA + IndexedDB + React Islands + 인터랙티브 플래시카드"** 라는 2030세대 개발자가 이상적으로 생각하는 스택을 설계했습니다.

하지만 50대 수험생은:

- PWA "홈 화면 추가" 개념을 모르는 경우가 대다수
- 작은 터치 타겟에 불편함
- 오프라인 학습보다 **큰 화면 PC 학습** 선호 가능성
- 서비스 신뢰도(브랜드)를 매우 중시 — 신생 서비스 꺼림
- 결제도 앱 카드보다 **전화 상담 → 계좌이체** 선호 가능

**→ 우리의 기술 스택 선택이 타겟 사용자와 충돌할 가능성이 있습니다.**

**C. 2차가 진짜 관문이다**

합격률 5.9%의 2차가 수험생이 가장 돈을 쓰는 영역입니다. 그런데 우리 설계는 **"1차 3과목 + 2차"를 동등하게 다룹니다.** 기획 균형을 재고해야 합니다. 2차에 80% 역량을 집중하고, 1차는 보조로 가는 전략이 수요와 더 잘 맞을 수 있습니다.

**D. 기존 학원이 강력하다**

박문각, 에듀플러스, 메가스터디가 이미 시장을 장악하고 있습니다. 이들은:

- **국비지원 (내일배움카드)**을 활용해 실질 수강료를 대폭 낮춤
- 오프라인 + 온라인 강의 모두 제공
- 합격자 후기/브랜드 자산 축적
- 전담 강사진

우리의 차별화 포인트 "Graph RAG + 암기법 자동 매칭"이 **이들 대비 연 30~50만 원의 가치를 제공하는가?** 이 질문에 대한 근거 있는 답이 없습니다.

## 2️⃣ Opus 4.7이 지적하는 7대 근본 누락 (치명도 순)

### 🔴 LEVEL 1: 사업 존립 위협 (4개)

#### ① 비즈니스 모델 자체가 없다

**현재 상태:** 구독/일시불/프리미엄 중 무엇인지 미정. 가격대 미정. 무료 체험 정책 미정.

**왜 심각한가:** 16주 개발 후 "얼마에 팔지"를 그때 정한다? 그러면 과금 플로우, 무료/유료 경계, 결제 시스템 통합이 전부 나중에 붙는 **패치 작업**이 됩니다. 상용 서비스의 생명선인 **수익 구조를 나중에 붙이는 것**은 구조적 실패 패턴입니다.

**Opus 권고:**

```
Phase 0 착수 전에 결정할 것:
- 가격: 월 2만 원? 일시불 30만 원? Freemium?
- 무료 vs 유료 경계: 기출 풀이만 무료, 암기법/대시보드는 유료?
- 환불 정책: 7일 이내 전액? 일부?
- 결제 수단: 토스페이먼츠? 카카오페이? 계좌이체 병행?
- 프로모션: 합격 시 환급? 추천인?
```

이것이 정해지지 않으면 DB 스키마의 `user_progress.subscription_tier`, 결제 이벤트, 기능 권한 게이팅이 모두 **임시방편**이 됩니다.

#### ② 법적 리스크 평가 전무

현재까지 이 주제는 단 한 번도 논의되지 않았습니다. 그러나 이것이 서비스 런칭 자체를 막을 수 있습니다.

**잠재 문제:**

| 리스크         | 근거                                                                                                        | 영향                       |
| -------------- | ----------------------------------------------------------------------------------------------------------- | -------------------------- |
| 교재 저작권    | 「농업재해보험 손해평가 이론과 실무」는 상업 서적. Graph 구조화 후 AI 학습/해설 생성이 2차 저작물 침해 가능 | 출판사 소송 시 서비스 중단 |
| 기출문제 활용  | Q-Net 공식 기출은 공개되나, 문제은행 사업자 권리와의 충돌 가능성                                            | 저작권 분쟁                |
| 학원법 저촉    | "합격 보장", "적중률 99%" 등의 광고는 학원의 설립·운영 및 과외교습에 관한 법률 저촉 가능                    | 광고 중단, 과징금          |
| 개인정보보호법 | 학습 이력, 약점 분석은 민감 정보. PIPA 준수 필요                                                            | 5억 이하 과태료            |
| 전자상거래법   | 콘텐츠 판매 시 7일 청약철회 의무                                                                            | 환불 거부 시 시정명령      |
| 표시광고법     | "AI가 맞혀준 문제" 같은 광고는 실증 자료 필요                                                               | 공정위 시정명령            |

**Opus 권고:**

Phase 0 착수 전에 **IT 전문 변호사 자문 1회** 받는 것이 비용($100~300) 대비 절대적으로 가치 있습니다. 특히 **교재 저작권 문제**는 서비스의 존립 자체를 결정합니다.

대안 경로:

- **교재 출판사와 라이선스 계약** 체결 시도 (협업 모델)
- 또는 교재 대신 **법제처 원문 + 기출 + 자체 제작 해설**로만 구성 (교재 우회)
- 후자는 기획 전면 수정. 교재 835쪽 분석 자산의 상당 부분을 재활용 불가

#### ③ 시장 검증(PMF) 부재

지금까지의 모든 설계는 **진산님의 가설**에 기반합니다:

- "수험생은 혼동 유형별 암기법을 원할 것이다"
- "Graph RAG 해설이 단순 해설보다 유용할 것이다"
- "PWA 오프라인 학습이 매력적일 것이다"

이 가설들 중 **실제 수험생을 만나서 검증된 것은 0개**입니다.

**Opus 권고:**

Phase 0 착수 전에 **수험생 인터뷰 5~10명**을 진행해야 합니다.

```
인터뷰 필수 질문:
1. 현재 어떤 서비스/학원으로 공부하시나요?
2. 월 평균 학습 비용은 얼마인가요?
3. 가장 어려운 과목은? 가장 불만족스러운 학습 방식은?
4. 특정 기능(예: 혼동 유형별 암기법)이 있다면 월 OO원 낼 의향?
5. 스마트폰으로 공부하시나요, PC로 하시나요?
```

이 질문들의 답이 우리 가설과 다르면, **16주 설계의 30~50%를 수정**해야 할 수 있습니다. 수정 비용이 착수 후에는 기하급수적으로 증가합니다.

#### ④ 운영 경제성 미검증

**현재까지 논의된 비용:** Claude API 파이프라인 1회 실행 $2 정도.

**미논의된 비용:**

```
월간 운영비 (사용자 1,000명 기준 추정):
- Cloudflare Workers Paid: $5
- D1 쿼리 비용 (월 500만 쿼리): ~$5
- Vectorize 쿼리 비용: ~$3
- R2 저장 비용: ~$2
- Claude API (사용자당 월 20회 AI 튜터, 1,000명):
    Haiku 기준 ~$50~100
- Workers AI (임베딩): ~$10
- 도메인/SSL: $10/월
- 모니터링(Sentry Free, PostHog Free): $0
- 이메일(Resend Free): $0
- ─────────────
- 합계 추정: 월 $85~135 (약 월 12~18만 원)
```

사용자 10,000명으로 스케일 시:

```
- Claude API가 병목: ~$500~1,000/월 (약 70~140만 원)
- D1/Vectorize 쿼리 비용 비례 증가: ~$50~100/월
- 합계: 월 $600~1,200 (약 85~170만 원)
```

**손익분기점 분석 (월 2만 원 구독 가정):**

- 월 18만 원 비용 커버: 유료 사용자 9명
- 월 170만 원 비용 커버: 유료 사용자 85명
- **진산님 인건비 월 300만 원까지 고려하면:** 유료 사용자 230명 필요

1만 명 시장에서 230명 유료 = **시장 점유율 2.3%**. 달성 가능성은 있으나, **마케팅 전략 없이는 불가능**합니다.

**Opus 권고:**

기획 착수 전 운영비 모델 스프레드시트를 만들어야 합니다. Claude API 비용이 선형적으로 증가하므로 AI 튜터 기능에 **월 쿼리 상한**을 걸어야 할 수도 있습니다.

### 🟠 LEVEL 2: 런칭 저해 요인 (3개)

#### ⑤ 보안 위협 모델링 (Sentinel 관점 누락)

현재 CLAUDE.md에 "Sentinel 페르소나"가 정의되어 있지만, **실제 위협 모델링 문서**는 작성되지 않았습니다.

**예상 위협 7가지:**

| 위협             | 공격 시나리오                            | 완화                                  |
| ---------------- | ---------------------------------------- | ------------------------------------- |
| 콘텐츠 스크래핑  | 경쟁사가 우리 Graph/기출 해설 크롤링     | Cloudflare Bot Management, Rate Limit |
| 계정 탈취        | 비밀번호 재사용 공격                     | 2FA 옵션, 이상 로그인 감지            |
| JWT 탈취         | XSS로 토큰 탈취                          | HttpOnly 쿠키, CSP 헤더               |
| 결제 사기        | 가짜 카드로 구독 후 환불                 | 토스페이먼츠 3D Secure                |
| AI 프롬프트 주입 | AI 튜터 질문에 시스템 프롬프트 유출 유도 | 프롬프트 Sanitization, Output Filter  |
| 관리자 CMS 침입  | admin 라우트 무차별 대입                 | Cloudflare Access, IP 제한            |
| DDoS             | 경쟁사 악의적 공격                       | Cloudflare 기본 방어 + Rate Limit     |

**Opus 권고:** Phase 0에 **THREAT_MODEL.md** 작성 필수.

#### ⑥ 관찰 가능성(Observability) 인프라 부재

현재 계획에는 Sentry, PostHog, Mixpanel 같은 분석 도구가 **언급조차 없습니다.**

**이것이 없으면:**

- 사용자가 어디서 이탈하는지 모름
- 어떤 기능이 실제 사용되는지 모름
- 에러가 프로덕션에서 발생해도 인지 못 함
- A/B 테스트 불가능 → 개선 가설 검증 불가

**Opus 권고:** Phase 0 인프라에 추가.

```
필수 통합 (전부 무료 티어):
- Sentry (에러 추적): 월 5,000 이벤트 무료
- PostHog Cloud (제품 분석): 월 100만 이벤트 무료
- Cloudflare Analytics (기본 제공)
- Wrangler Tail (실시간 로그)
```

#### ⑦ 접근성(A11y) 법적 의무 미논의

**장애인차별금지법**은 **공공성이 있는 서비스**에 접근성 보장 의무를 부과합니다. 자격증 시험 준비 서비스는 공공성 주장 가능 영역입니다.

또한 **주 사용자가 50대**라는 사실은 **돋보기 사용자, 시력 저하자, 청력 저하자**가 상당 비율임을 의미합니다.

**미적용 시 위험:**

- 장애인 차별 진정 접수 시 법적 분쟁
- 50대 사용자 이탈률 급증

**Opus 권고:**

- WCAG 2.1 AA 기준 준수 목표
- Phase 2에 **접근성 체크리스트** 추가 (최소 요구사항):
  - 모든 이미지에 alt 속성
  - 폰트 크기 조절 가능 (120%, 150%, 200%)
  - 키보드만으로 모든 기능 사용 가능
  - 색상 대비 WCAG AA (4.5:1)
  - 스크린 리더 호환 (ARIA 레이블)

## 3️⃣ 50대 주 사용자 발견이 흔드는 설계 전제

이것은 단순 "접근성 고려"가 아닙니다. **전체 기술 스택과 UX 철학의 재검토**를 요구합니다.

### 현재 전제 vs 50대 수험생 현실

| 현재 전제               | 50대 현실                     | 재검토 방향                        |
| ----------------------- | ----------------------------- | ---------------------------------- |
| 모바일 퍼스트 PWA       | **데스크탑 중심 학습** 가능성 | 반응형 but PC UX를 1순위로         |
| IndexedDB 오프라인      | "오프라인 학습" 개념 생소     | 단순 "지하철에서도 됨" 정도로 홍보 |

| 다크모드                | 사용률 낮음                   | Phase 3로 후순위                   |
| 카카오/구글 소셜 로그인 | 이메일/비밀번호 선호          | 둘 다 제공하되 이메일 우선         |

### Opus 권고: "듀얼 UX 전략"
