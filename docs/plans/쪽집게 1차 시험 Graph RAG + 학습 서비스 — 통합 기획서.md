├─────────────────────────────────────────────────────────────────────┤

## 3. 과목별 Graph RAG 전략

### 3.1 상법 보험편 — 정방향 법조문 Graph

#### 데이터 소스

```
[1차 소스: 법제처 상법 원문]
├── 제4편 보험 (제638~730조)
│   ├── 제1장 통칙 (제638~664조) — 보험계약 일반
│   ├── 제2장 손해보험
│   │   ├── 제1절 통칙 (제665~679조)
│   │   ├── 제2절 화재보험 (제680~681조)
│   │   ├── 제3절 운송보험 (제682~692조)
│   │   ├── 제4절 해상보험 (제693~718조)
│   │   └── 제5절 책임보험 (제719~724조)
│   └── 제3장 인보험 (제725~730조)
│
[2차 소스: 기출 275문항]
│
[3차 소스: 판례/해석 (보충)]
```

#### 노드 설계

| 노드 타입 | 설명                                      | 예상 수량 |
| --------- | ----------------------------------------- | --------- |
| LAW       | 조문 원문 (조/항 단위)                    | ~100      |
| CONCEPT   | 보험 원리/개념 (고지의무, 피보험이익 등)  | ~40       |
| TERM      | 용어 정의 (보험자, 보험계약자, 보험료 등) | ~30       |
| EXCEPTION | 예외 규정 ("단, ~의 경우")                | ~20       |

#### 엣지 설계

| 엣지 타입    | 의미              | 예시                                 |
| ------------ | ----------------- | ------------------------------------ |
| GOVERNED_BY  | 개념 → 조문 근거  | 고지의무 → 제651조                   |
| PREREQUISITE | 조문 간 선행 관계 | 고지의무위반(651) → 해지권(652)      |
| EXCEPTION    | 원칙 → 예외       | 고지의무 → "중대한 과실 없으면" 예외 |
| CROSS_REF    | 조문 간 준용/참조 | 제664조 → 제638조 준용               |

#### 매직 넘버

| 카테고리 | 예시                                                    | 수량 |
| -------- | ------------------------------------------------------- | ---- |
| 기간     | 고지의무 위반 해지: "안 날로부터 1월, 체결일로부터 3년" | ~10  |
| 비율     | 초과보험: 보험가액 초과 시 "초과 부분 무효"             | ~5   |
| 금액     | 소멸시효: 보험금 2년, 보험료반환 3년                    | ~5   |

#### 혼동 위험 포인트

```
[danger] 기간 혼동:
  - 고지의무 위반: "1월 + 3년" vs 위험변경통지: "지체없이"
  - 보험금 시효 2년 vs 보험료반환 시효 3년

[danger] 긍부정 함정:
  - "고지의무는 알고 있는 사실을 알려야" (O)
  - "고지의무는 모든 사실을 알려야" (X — "중요한 사항"만)

[warn] 준용 관계:
  - 해상보험 → 손해보험 통칙 준용 범위
  - 인보험 → 손해보험 규정 일부 준용
```

### 3.2 농어업재해보험법령 — 법령 계층 Graph + 개정 추적

#### 데이터 소스

```
[1차 소스: 법령 원문]
├── 농어업재해보험법 (본법) — truth_weight: 10
├── 농어업재해보험법 시행령 — truth_weight: 9
└── 손해평가요령 (농식품부 고시) — truth_weight: 8

[2차 소스: 기출 275문항]

[3차 소스: 2차 교재 (835쪽) — 겹치는 부분 공유]
```

#### 3단계 위계 Graph

```
┌─────────────────────────────────────────────┐
│  법률 (truth_weight=10)                      │
│  "농어업재해보험법 제11조의3"                │
│  → 손해평가사의 업무                         │
├─────────────────────────────────────────────┤
│  시행령 (truth_weight=9)                     │
│  "시행령 제12조의5"                          │
│  → 1차 시험 면제 조건 (시행령에서 구체화)    │
├─────────────────────────────────────────────┤
│  고시 (truth_weight=8)                       │
│  "손해평가요령 제8조"                        │
│  → 손해평가반 구성 세부기준                  │
└─────────────────────────────────────────────┘
         ↕ GOVERNED_BY 엣지
```

#### 개정 추적 (시험 출제 최우선)

```
주요 개정 이력:
- 2014.6.3: 손해평가사 자격제도 신설
- 2016.12.2: 손해방지의무 개정
- 2020.2.11: 명의대여 금지 신설
- 2020.12.8: 분과위원회 규정 신설
- 2021.11.30: 기본계획 수립 신설
- 2023.3.28: 이의신청 절차 신설
- 2023.10.31: 수산업 심의회 규정 신설
- 2025.8.14: 의견청취 의무 개정

→ 2023~2025 개정 사항 = exam_priority 최고
```

#### 2차 교재와의 공유 영역

```
겹치는 노드 (공유 가능):
├── 보험의 목적, 보험가액, 보험금 — 1과목 + 2차 공통
├── 손해평가 요령 전체 — 2과목 + 2차 공통
├── 자기부담비율, 경작불능 기준 — 2과목 + 2차 공통
└── 보장방식별 약관 내용 — 2과목 + 2차 공통

공유 효과:
"1차에서 외운 법조문 → 2차에서 실무 적용"
동일 노드를 다른 관점(법령 vs 실무)에서 학습 = 자연스러운 반복
```

### 3.3 농학개론 — 역공학 Graph RAG ★★★★★

#### 핵심 전략: 삼각 수렴법

```
[소스 A: 이론서 (스캔)]        [소스 B: 기출 275문항]        [소스 C: 웹 공개자료]
  커리큘럼 골격 추출              개념별 출제 빈도 태깅        해설 보강 + 팩트체크
         \                              |                         /
          └──────────────────────────────┼────────────────────────┘
                                         ▼
                         [통합 Knowledge Graph]
                         - A가 뼈대(구조)
                         - B가 가중치(중요도)
                         - C가 살(해설)
```

#### 3단계 역공학 프로세스

**Stage 1: 기출 파싱 + 토픽 클러스터링**

```
[기출 275문항 입력]
    │
    ▼
[Claude API: 문항별 메타데이터 추출]
    │  - 테스트하는 개념 (예: "벼의 이앙시기")
    │  - 암기 유형 (정의형/나열형/수치형/조건형...)
    │  - 혼동 유형 (숫자/날짜/긍부정/예외...)
    │  - 난이도 (상/중/하)
    ▼
[토픽 클러스터링]
    │  275문항 → 약 60~80개 토픽 클러스터
    │  예: "토양pH", "접목법", "광합성", "과수 전정"
    ▼
[출제 빈도 히트맵]
    │  각 토픽별 "9년간 N회 출제" 집계
    │  → exam_frequency 필드
```

**Stage 2: 커리큘럼 트리 재구성 (이론서로 보강)**

```
[이론서 목차 스캔]
    │  재배학: 기원/품종/환경/기술/병충해
    │  원예학: 채소/과수/화훼
    ▼
[목차 → LV1/LV2/LV3 매핑]
    │  LV1: 재배학 | 원예작물학
    │  LV2: 재배환경 | 재배기술 | 병충해
    │  LV3: 토양 | 온도 | 광 | 수분 | 대기
    ▼
[기출 토픽을 커리큘럼 트리에 배치]
    │  토픽 "토양pH" → LV1:재배학 > LV2:재배환경 > LV3:토양
    │  토픽 "사과 전정" → LV1:원예작물학 > LV2:과수 > LV3:사과
    ▼
[미출제 영역 식별]
    │  커리큘럼에 있지만 기출에 없는 토픽 = "출제 가능 미지 영역"
    │  → 경고 태그 부착
```

**Stage 3: 해설 보강 (웹 자료 + Claude 생성)**

```
[각 토픽 노드에 해설 추가]
    │
    ├── 농촌진흥청 작물별 재배 매뉴얼 (무료 공개)
    ├── 대학교재 발췌 (이론서 스캔 해당 부분)
    ├── Claude 웹검색 결과
    │
    ▼
[해설 품질 검증]
    │  기출 정답과 일치하는지 삼각 검증
    │  불일치 시 → flagged → 인간 검수
```

#### 농학개론 암묵적 커리큘럼 (예상)

```
농학개론
├── 1편: 재배의 기원과 현황
│   ├── 재배식물의 기원과 발달
│   └── 작물의 분류 (식량/공예/사료/특용)
│
├── 2편: 작물의 품종과 육종
│   ├── 품종의 개념
│   ├── 육종 방법 (교잡/돌연변이/배수체)
│   └── 신품종 보급
│
├── 3편: 재배환경 ★출제 비중 최고
│   ├── 토양환경 (pH, 양분, 토성, 토양수)
│   ├── 온도환경 (적산온도, 생육적온, 한계온도)
│   ├── 광환경 (광합성, 광주기성, 장/단일식물)
│   ├── 수분환경 (증산, 관개, 배수)
│   ├── 대기환경 (CO₂, O₂, 바람)
│   └── 상적발육과 환경 (춘화, 화아분화)
│
├── 4편: 재배기술
│   ├── 작부체계 (윤작, 간작, 혼작)
│   ├── 종묘 (종자 발아, 육묘)
│   ├── 생장조절제 (옥신, 지베렐린, 에틸렌)
│   └── 재배관리 (시비, 관수, 중경, 제초)
│
├── 5편: 병충해 및 재해 ★2차 교재와 연결
│   ├── 병충해 (세균/진균/바이러스/해충)
│   ├── 잡초 (분류, 방제)
│   └── 기상재해 (동해, 상해, 한해, 풍해, 우박, 도복)
│       └── → 2차 교재 "각종 재해" 섹션과 직결
│
└── 6편: 원예작물
    ├── 채소류 (엽근채/과채/엽경채)
    ├── 과수류 (핵과류/인과류/감귤류/장과류)
    └── 화훼류 (1·2년초/구근/숙근초)
```

#### 농학개론 매직 넘버 (기출에서 추출 예상)

| -------- | ------------------------------------- | -------------------------- |

## 4. 통합 DB 스키마 확장

기존 2차용 6개 테이블을 1차까지 포괄하도록 확장.
테이블 구조는 동일하되, `exam_scope` 필드를 추가.

### 4.1 확장 필드

```sql
-- knowledge_nodes에 추가
ALTER TABLE knowledge_nodes ADD COLUMN exam_scope TEXT DEFAULT '2nd';
-- '1st_sub1' | '1st_sub2' | '1st_sub3' | '2nd' | 'shared'
-- 'shared' = 1차-2차 공유 노드

-- exam_questions에 추가
ALTER TABLE exam_questions ADD COLUMN exam_type TEXT DEFAULT '2nd';
-- '1st' | '2nd'
ALTER TABLE exam_questions ADD COLUMN subject TEXT;
-- '상법보험편' | '농어업재해보험법령' | '농학개론' | '손해평가이론실무'

-- constants에 추가
ALTER TABLE constants ADD COLUMN exam_scope TEXT DEFAULT '2nd';
-- 1차-2차 공유 constants 식별
```

### 4.2 과목 간 노드 공유 규칙

```
[공유 가능 노드 식별 로직]

1과목(상법) ↔ 2과목(법령):
  - 보험계약, 보험료, 보험금, 보험가액 → exam_scope='shared'
  - 고지의무, 위험변경통지 → exam_scope='shared'

2과목(법령) ↔ 2차(손해평가):
  - 손해평가요령 전체 → exam_scope='shared'
  - 자기부담비율, 경작불능 기준 → exam_scope='shared'
  - 보장방식별 약관 → exam_scope='shared'

3과목(농학) ↔ 2차(손해평가):
  - 병충해/재해 종류 → exam_scope='shared'
  - 작물 생육특성 (수확적기, 생장일수) → exam_scope='shared'
  - 재배환경 피해 (동해, 풍해, 우박) → exam_scope='shared'

## 5. 혼동 유형 × 암기법 매칭 엔진

### 5.1 혼동 유형 자동 감지 (Graph 데이터 기반)

| 혼동 유형      | 감지 소스                           | 감지 로직                            |
| -------------- | ----------------------------------- | ------------------------------------ |
| 숫자/수치 혼동 | `constants` confusion_level=danger  | 같은 category인데 값이 다른 쌍       |
| 날짜/기간 혼동 | `constants` category=date           | 같은 문맥의 서로 다른 기한           |
| 긍부정 함정    | `knowledge_nodes` type=LAW          | "~하여야" vs "~할 수 있다" 자동 분류 |
| 예외 함정      | `knowledge_edges` type=EXCEPTION    | 원칙-예외 쌍 자동 추출               |
| 절차 순서      | `knowledge_edges` type=PREREQUISITE | 3개+ 체인 = 순서 혼동 위험           |
| 작물간 교차    | `knowledge_edges` type=DIFFERS_FROM | 같은 조사명 + 다른 기준 쌍           |
| 법조문 준용    | `knowledge_edges` type=CROSS_REF    | 준용 범위 혼동                       |
| 나열 항목 누락 | `knowledge_nodes` + 기출분석        | "N가지" 나열 중 빠뜨리기 쉬운 항목   |

### 5.2 암기법 매칭 매트릭스

| 혼동 유형        | 1순위 암기법               | 2순위             | 자동 생성 컨텐츠     |
| ---------------- | -------------------------- | ----------------- | -------------------- |
| 숫자/수치        | Peg System + 대비표        | 연상법            | 숫자 대비 플래시카드 |
| 소수점 계수      | 숫자분해법 + 반복연산      | 산식카드          | 인터랙티브 계산기    |
| 보험금 비율 세트 | 패턴인식 + Chunking        | 표 암기           | 비율표 시각화        |
| 절대 날짜        | Memory Palace + 달력       | 스토리법          | 연간 달력 타임라인   |
| 상대 기간        | Peg System                 | 카테고리 그룹화   | 기간별 그룹 카드     |
| 시점 조건부 분기 | 분기 플로우차트            | 조건부 스토리     | 인터랙티브 분기트리  |
| 긍부정 전환      | OX 반전 훈련               | 긍부정 플래시카드 | OX 랜덤 전환 문제    |
| 예외 함정        | 대비 스토리                | 예외 강조 카드    | 원칙+예외 대비 카드  |
| 나열 항목        | 두문자어(Acronym)          | Memory Palace     | AI 두문자어 생성     |
| 절차 순서        | Memory Palace + 플로우차트 | 두문자어          | SVG 플로우차트       |
| 작물간 교차      | 교차 비교표                | 차이점 강조       | 매트릭스 위젯        |
| 법조문 준용      | 관계도 시각화              | 스토리법          | Graph 서브뷰         |

### 5.3 개인화 로직

```
[사용자 오답 패턴 분석]
