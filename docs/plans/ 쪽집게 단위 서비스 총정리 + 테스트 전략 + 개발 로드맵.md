## 1. 테스트 철학

### 시험 서비스의 특수성

```
일반 서비스:  버그 → 불편 → 패치
시험 서비스:  오답 → 불합격 → 인생 → 신뢰 파괴 → 서비스 사망

따라서:
1. 모든 "정답"이 포함된 출력은 100% 정확도가 목표
2. 100% 달성 불가능한 영역은 "신뢰도 라벨"을 반드시 표시
3. 테스트를 통과하지 않은 모듈은 절대 다음 모듈의 입력이 될 수 없음
4. 인간 검수를 거치지 않은 데이터는 사용자에게 노출 금지
```

### 테스트 계층

```
┌─────────────────────────────────────────────────────────────────────┐
│  Level 5: 사용자 수용 테스트 (UAT)                                 │
│  "실제 수험생이 써보고 문제없나?"                                  │
├─────────────────────────────────────────────────────────────────────┤
│  Level 4: 통합 테스트 (Integration)                                │
│  "모듈 A의 출력이 모듈 B에 들어갔을 때 정상인가?"                 │
├─────────────────────────────────────────────────────────────────────┤
│  Level 3: 정확도 검증 (Accuracy)                                   │
│  "기출 정답과 일치하는가? 산식 계산이 맞는가?"                     │
├─────────────────────────────────────────────────────────────────────┤
│  Level 2: 기능 테스트 (Functional)                                 │
│  "입력 X를 넣으면 출력 Y가 나오는가?"                              │
├─────────────────────────────────────────────────────────────────────┤
│  Level 1: 단위 테스트 (Unit)                                       │
│  "함수 f(x)가 올바른 값을 반환하는가?"                             │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. Layer 1: 데이터 수집

### M01. PDF 텍스트 추출기

| 항목     | 내용                                              |
| -------- | ------------------------------------------------- |
| **역할** | 교재 PDF(835쪽) → 페이지별 raw text 추출          |
| **기술** | pdfplumber (Python subprocess)                    |
| **입력** | PDF 파일 경로                                     |
| **출력** | `{page: number, text: string, tables: Table[]}[]` |

| 테스트 ID | 테스트 항목        | 통과 기준                      | 테스트 방법                |
| --------- | ------------------ | ------------------------------ | -------------------------- |
| M01-T01   | 텍스트 추출 완전성 | 835페이지 100% 추출            | 페이지 수 카운트           |
| M01-T02   | 한글 인코딩 정확도 | 깨진 문자 0건                  | 랜덤 50페이지 육안 검증    |
| M01-T03   | 표(Table) 추출     | 주요 표 10개 중 8개+ 정상 추출 | 수동 대조                  |
| M01-T04   | 특수문자/수식      | ×, ÷, %, ≥, ≦ 등 정상          | 산식 포함 페이지 10개 검증 |
| M01-T05   | 성능               | 835쪽 전체 추출 5분 이내       | 시간 측정                  |

**품질 게이트:** M01-T01~T04 전부 통과 시에만 M06(섹션 분리기)로 진행

**품질 게이트:** 조문 누락 0건 확인 후 M07(배치 프로세서)로 진행

### M03. 기출문제 파서

| 항목     | 내용                                                                        |
| -------- | --------------------------------------------------------------------------- |
| **역할** | 기출 PDF → 문항별 구조화 JSON                                               |
| **기술** | pdfplumber + 정규식 + Claude API 보조                                       |
| **입력** | 기출 PDF (1~11회, 1차+2차)                                                  |
| **출력** | `{id, year, round, subject, q_no, stem, choices[], answer, explanation?}[]` |

| 테스트 ID | 테스트 항목    | 통과 기준                              | 테스트 방법           |
| --------- | -------------- | -------------------------------------- | --------------------- |
| M03-T01   | 문항 수 정합성 | 1차: 과목당 25 × 11회 = 275, 전체 825+ | 카운트                |
| M03-T02   | 정답 정확도    | **100%** (공식 정답과 불일치 0건)      | 전수 대조             |
| M03-T03   | 선지 파싱      | 4지선다 4개 선지 정확 분리             | 랜덤 50문항 수동 검증 |
| M03-T04   | 지문 완전성    | 지문 잘림/누락 0건                     | 랜덤 30문항 원본 대조 |
| M03-T05   | 특수문자/수식  | 산식 포함 문항 정상 파싱               | 계산형 문항 전수 검증 |

**품질 게이트:** M03-T02(정답 100%)는 절대 타협 불가. 불일치 1건이라도 있으면 원인 규명 후 재실행.

### Hard Stop 규칙

```
다음 조건 중 하나라도 해당되면 전체 프로젝트 Hard Stop:

1. 기출 정답과 불일치하는 해설이 사용자에게 노출된 경우

→ 즉시 해당 기능 비활성화 + 원인 규명 + 재검증 후에만 복원
```
