# 쪽집게(ThePick) — 구현 재정립서

> 8개 기획문서를 통합 분석하여 개발 실행을 위한 단일 진실의 원천(Single Source of Truth)으로 재정립
>
> 원본 문서: 통합 기획서, 파이프라인 설계서, 교재 분석서, 1차 시험 기획서, GAP 분석서, 단위 서비스 총정리, SQL 스키마 2종, 서비스맵 HTML
>
> 작성일: 2026-04-11

## 1. 프로젝트 정의

**쪽집게(ThePick)** — 손해평가사 자격시험(1차+2차) AI 학습 서비스

| 항목           | 값                                                                                            |
| -------------- | --------------------------------------------------------------------------------------------- |
| 도메인         | 손해평가사 자격시험 (1차 객관식 3과목 + 2차 실기)                                             |
| 핵심 가치      | 835쪽 교재 + 기출 825문항을 Graph RAG로 구조화 → AI가 정확한 해설/문제 생성                   |
| 차별점         | 단순 암기 앱이 아닌, 지식 그래프 기반 조건부 라우팅 + 룰 엔진 산식 연산 + 혼동 유형 자동 감지 |
| 합격 기준(1차) | 매 과목 40점+, 전 과목 평균 60점+                                                             |
| 합격 기준(2차) | 100% 서술형/계산형                                                                            |

### 1차 시험 3과목

| 과목               | 핵심 전략                               | 데이터 소스                          | 난이도 |
| ------------------ | --------------------------------------- | ------------------------------------ | ------ |
| 상법 보험편        | 정방향 법조문 Graph (제638~730조)       | 법제처 원문 + 기출 275문항           | ★★☆    |
| 농어업재해보험법령 | 법령 계층 Graph + 개정 추적             | 본법+시행령+고시 + 기출 275문항      | ★★★    |
| 농학개론           | 역공학 Graph RAG (기출→지식체계 재구성) | 기출 275문항 + 이론서 스캔 + 웹 자료 | ★★★★★  |

### 2차 시험

| 과목                 | 핵심                    | 데이터 소스                                                 |
| -------------------- | ----------------------- | ----------------------------------------------------------- |

## 3. 3계층 데이터 아키텍처

```
[정밀 계층] D1: constants 테이블
  수치/날짜/임계값 → 룰 엔진이 DB에서 직접 조회, LLM 추론 금지

[구조 계층] D1: knowledge_nodes + edges + formulas
  개념관계/조건분기/절차흐름 → 메타데이터 필터 후 검색

[맥락 계층] Vectorize: 임베딩
  해설/부연설명/사례 → 벡터 유사도 검색
```

### 방어 장치 4종

| 장치                 | 규칙                                                                                      |
| -------------------- | ----------------------------------------------------------------------------------------- |
| Truth Weight         | LAW(10) > FORMULA(8) > INVESTIGATION(7) > CONCEPT(5) > TERM(3) — 충돌 시 높은 가중치 우선 |
| Temporal Graph       | UPDATE 금지. 개정 시 신규 노드 + SUPERSEDES 엣지. 구 엣지 is_active=0                     |
| Graceful Degradation | 유사도 < 0.60 → 해설 거부 + "교재 O장 O절 참고" 안내                                      |
| Constants 직접 조회  | LLM에게 숫자 추론 절대 금지. DB 쿼리로만 가져옴                                           |

## 4. Graph RAG 4-Level 메타데이터 스키마

```
LV1: 보장방식 (Insurance Type) — 8종
 └── LV2: 품목/작물군 (Crop Category) — 70+종
      └── LV3: 조사 종류 (Investigation Type) — 16종
           └── LV4: 구성 요소 (시기/방법/산식/조건/상수)
```

### 노드 유형 (7종)

| 타입          | truth_weight | 예상 수량               |
| ------------- | ------------ | ----------------------- |
| LAW           | 10           | ~150 (상법93 + 법령70+) |
| FORMULA       | 8            | ~125                    |
| INVESTIGATION | 7            | ~20                     |
| INSURANCE     | 6            | ~8                      |
| CROP          | 6            | ~70                     |
| CONCEPT       | 5            | ~430                    |
| TERM          | 3            | ~240                    |

### 엣지 유형 (12종)

APPLIES_TO, REQUIRES_INVESTIGATION, PREREQUISITE, USES_FORMULA, DEPENDS_ON, GOVERNED_BY, DEFINED_AS, EXCEPTION, TIME_CONSTRAINT, SUPERSEDES, SHARED_WITH, DIFFERS_FROM, CROSS_REF

### 통합 규모 추정

| 범위                  | 노드       | 엣지       | 상수     | 산식     |
| --------------------- | ---------- | ---------- | -------- | -------- |
| 1차 3과목             | ~670       | ~1,400     | ~100     | ~30      |
| 2차 손해평가          | ~620       | ~1,500     | ~70      | ~105     |
| 공유 노드 (중복 제거) | -150       | -300       | -20      | -10      |
| **합계**              | **~1,140** | **~2,600** | **~150** | **~125** |

## 5. DB 스키마 (9개 테이블)

기본 6개 테이블은 `Graph rag schema v1.sql`에 정의:

| 테이블             | 역할                 | 핵심 필드                                             |
| ------------------ | -------------------- | ----------------------------------------------------- |
| `knowledge_nodes`  | 지식 노드            | type, lv1~lv3, truth_weight, version_year, status     |
| `knowledge_edges`  | 노드 간 관계         | edge_type, condition, is_active                       |
| `formulas`         | 산식 (룰 엔진용)     | equation_template, variables_schema, expected_inputs  |
| `constants`        | 매직 넘버 레지스트리 | category, value, numeric_value, unit, confusion_level |
| `revision_changes` | 개정 이력 추적       | change_type, exam_priority, before/after_value        |
| `exam_questions`   | 기출문제 (시간축)    | valid_from/until, superseded_by, status               |

1차 확장 3개 테이블은 `Schema extension 1st exam.sql`에 정의:

| 테이블           | 역할                                        |
| ---------------- | ------------------------------------------- |
| `mnemonic_cards` | 암기법 매칭 (두문자어/연상법/Memory Palace) |
| `user_progress`  | 학습 진도 + FSRS 파라미터                   |
| `topic_clusters` | 농학개론 역공학용 토픽 클러스터             |

확장 필드: knowledge_nodes.exam_scope, exam_questions.exam_type/subject/topic_cluster/memorization_type/confusion_type, constants.exam_scope

## 7. 파싱 파이프라인 (교재 → 구조화 데이터)

### 8단계 파이프라인

```
Stage 1: PDF 835쪽 → pdfplumber → 페이지별 raw text
Stage 2: 정규식 + 구조 인식 → 절/항/호/목 단위 분리 + 표 추출
Stage 3: Claude API 배치 → Knowledge Contract JSON 초안
Stage 4: Ontology Lock + 스키마 검증 → 미등록 ID 거부/재수행
Stage 5: D1 INSERT (status='draft')
Stage 6: Graph Visualizer → 인간 검수 (draft → review → approved → published)
Stage 7: approved 노드만 Vectorize 임베딩
Stage 8: 학습자료 자동 생성
```

## 8. 혼동 유형 감지 + 암기법 매칭

### 8종 혼동 유형

| 유형           | 감지 소스                         | 예시                          |
| -------------- | --------------------------------- | ----------------------------- |
| 숫자/수치 혼동 | constants confusion_level=danger  | 경작불능 65% vs 분질미 60%    |
| 소수점 계수    | constants category=coefficient    | 단감 1.0115 vs 떫은감 0.9662  |
| 날짜/기간 혼동 | constants category=date           | 이앙한계일 7.31, 낙엽조사 6.1 |
| 긍부정 함정    | knowledge_nodes type=LAW          | "~하여야" vs "~할 수 있다"    |
| 예외 함정      | knowledge_edges type=EXCEPTION    | 원칙-예외 쌍                  |
| 절차 순서      | knowledge_edges type=PREREQUISITE | 3개+ 체인                     |
| 작물간 교차    | knowledge_edges type=DIFFERS_FROM | 사과 vs 포도 착과수조사       |
| 나열 항목 누락 | nodes + 기출분석                  | "N가지" 중 빠뜨리기 쉬운 항목 |

### 암기법 매칭 매트릭스

| 혼동 유형   | 1순위 암기법               | 생성 컨텐츠          |
| ----------- | -------------------------- | -------------------- |
| 숫자/수치   | Peg System + 대비표        | 숫자 대비 플래시카드 |
| 소수점 계수 | 숫자분해법 + 반복연산      | 인터랙티브 계산기    |
| 날짜/기간   | Memory Palace + 달력       | 연간 달력 타임라인   |
| 긍부정      | OX 반전 훈련               | OX 랜덤 전환 문제    |
| 예외        | 대비 스토리                | 원칙+예외 대비 카드  |
| 절차 순서   | Memory Palace + 플로우차트 | SVG 플로우차트       |
| 작물간 교차 | 교차 비교표                | 매트릭스 위젯        |
| 나열        | 두문자어(Acronym)          | AI 두문자어 생성     |

## 10. 1차-2차 통합 시너지

공유 노드 ~150개 (전체의 ~13%):

- 보험 기본 개념 (상법 ↔ 2차): ~30개
- 재해보험 법령 (법령 ↔ 2차): ~80개
- 작물 재배 특성 (농학 ↔ 2차): ~40개

**효과:** "1차 준비가 2차의 50%를 선행 학습하는 효과" — 동일 노드를 다른 관점(법령 vs 실무)에서 학습

## 14. Hard Rules (전 단계 적용, 11개)

```
1. UPDATE 금지: knowledge_nodes, formulas는 절대 UPDATE하지 않는다.
   개정 시 새 노드 생성 → SUPERSEDES 엣지 → 구 엣지 is_active=0

2. LLM 연산 금지: 수식 계산은 Formula Engine만 수행.
   LLM은 "어떤 산식을 적용할지" 판단만 한다.

3. Truth Weight 강제 정렬: RAG 결과를 LLM에 주입할 때
   LAW(10) > FORMULA(8) > INVESTIGATION(7) > CONCEPT(5) 순서.

4. Graceful Degradation: 유사도 < 0.60이면 해설 생성하지 않고
   "교재 O장 O절을 참고해 주세요"로 후퇴.

5. Constants 직접 조회: 수치/날짜/임계값은 DB 쿼리로만 가져온다.

6. 배치 순서 엄수: BATCH N 검증 없이 BATCH N+1 진행 금지.

7. 인간 검수 필수: AI 생성 데이터는 draft로만 적재.
   approved는 Graph Visualizer에서 인간 확인 후에만.

8. Ontology Lock: ontology-registry.json에 정의된 ID만 허용.
   미등록 ID → 거부 + 재수행 강제.

9. 동적 코드 실행 금지: equation_template 실행은 math.js AST 파서로만.
   동적 코드 실행 함수는 어떤 경우에도 사용하지 않는다.

10. Hairball 방지: Graph Visualizer는 서브그래프 단위만 렌더링.

11. 기출 정답 최우선: Graph 해설이 기출 공식 정답과 충돌 → flagged 처리.
```
