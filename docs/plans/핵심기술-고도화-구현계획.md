### 🟠 P1 — Phase 1 내 실행 (해당 Step 도달 시)

#### [HK-03] math.js 번들 최적화

**현재**: `create(all)`로 전체 math.js 로드. 번들 ~170KB.
**위험**: Workers 10MB 제한 + 50ms free-tier CPU에서 초기화 비용.
**보고서 근거**: "번들 크기가 Workers 한도를 압박하면 fparse로 교체 검토"

```
실행 계획:
1. 현재 번들 크기 측정
   - `wrangler deploy --dry-run --outdir dist` → dist 분석
   - math.js가 전체 API 번들의 몇 %인지 확인

2. 트리쉐이킹 시도
   - `create(all)` → `create({...})` 필요한 함수만 임포트
   - 사용 함수: add, subtract, multiply, divide, ceil, floor,
     round, max, min, abs, sqrt, pow, mod, parse, compile
   - 예상 절감: ~170KB → ~60KB

3. fparse 교체 평가 (번들 > 100KB 시)
   - fparse: ~15KB, AST 지원, 우리 산식의 함수 커버리지 확인
   - max(), min(), ceil() 지원 여부가 핵심
   - 보안: fparse에도 동일한 화이트리스트 적용 필요

4. 실행 시점: Workers 배포 전 (Phase 1 Week 9~10)
5. DEFCON: L3 (계산 엔진 변경)
```
