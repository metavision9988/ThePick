## ✅ 지금 "확실히 되는 것" (일상어)

1. **회원가입** — 이메일 + 강한 비밀번호로 가입
2. **유출 비밀번호 차단** — HIBP 로 실시간 조회
3. **로그인 / 로그아웃** — 쿠키 발급 + 파기
4. **로그인 상태 유지** — 15분마다 자동 갱신
5. **여러 기기 동시 로그인** — PC + 모바일 독립 세션
6. **무차별 대입 방어** — IP 당 5분 20회 제한
7. **이메일 열거 방어** — 가입 여부 추측 불가
8. **탈취 감지** — 훔친 refresh 재사용 시 자동 감지
9. **관리자 BAN** — 정지 즉시 세션 파기
10. **결제 알림 수신** — PG 서명 검증 + 중복 방지
11. **위조 결제 차단** — 서명 실패 시 거부
12. **공격 방어** — 거대 페이로드 CPU 공격 차단
13. **네트워크 오탐 방지** — 60초 유예 창
14. **30일 만료** — 장기 미사용 자동 재로그인

## 📌 판정

> **🟢 Step 1-5 진입 가능 — 엔진(기초공사) 전부 견고함 확인.**

### Step 1-5 방향 2갈래 (진산님 선택)

**(가) 교재 Graph 구축 트랙** (개발 로드맵 원본)

- 상법 보험편 법조문 + 기출 문제를 DB 에 넣는 작업
- 사용자 화면은 없으나 "진짜 콘텐츠"가 서버에 쌓이기 시작
- 진산님이 API 로 `SELECT * FROM knowledge_nodes` 같은 쿼리 직접 확인 가능
- 규모 크므로 여러 Step (1-5 ~ 1-7) 에 나뉨

**(나) 사용자 진도 관리 API** (엔진 통합 검증)

- 지금까지 만든 인증 엔진을 실제 보호 라우트(`/api/progress/*`)에 **처음 마운트**
- `require-auth` 미들웨어 실전 사용 → JWT 세션 전체 플로우 E2E 검증 완결
- 작은 규모 (1~2일), 시나리오 수 시연 가능

**제 권고**: **(나) 먼저** — 만든 인증 엔진을 실제로 "쓰는" 첫 사례. 이게 통과해야 (가) 의 교재 Graph 도 같은 엔진에 얹을 수 있음. 또한 **진산님이 눈으로 "로그인→보호 API 호출→데이터 반환"** 전체 플로우를 처음 볼 수 있는 지점.
