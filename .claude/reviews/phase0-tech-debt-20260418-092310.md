## 지금까지 놓쳐왔던 진짜 위협 TOP 10 (4-Pass도 미발견)

1. **[Backend D-1] Temporal Graph 보호 트리거 절반만 적용** ★ 서비스 사망급
   - `prevent_*_update` 트리거가 `knowledge_nodes` + `formulas` 2개만. `constants`, `revision_changes`, `exam_questions`는 미보호
   - 2년차에 주니어 개발자가 `UPDATE constants SET numeric_value=0.60` 실행 → Temporal 이력 전부 소실 → 사용자 집단 환불 소송

2. **[Performance P-C1] mathjs 15 전체 번들 포함** (160KB gzipped)
   - `import { create, all } from 'mathjs'; const math = create(all);` 패턴
   - 선택적 `parseDependencies, typedDependencies, addDependencies, ...` 패턴으로 **110~130KB 절감 가능**
   - 현재 apps/api Workers 번들이 콜드스타트 parse 400~800ms

3. **[Performance P-C2] PBKDF2 600k × Workers Free 티어 구조적 불가능**
   - 10ms CPU 한도 < 200~400ms 연산. Free 배포 시 **로그인 전건 실패**
   - ADR-005 트리거가 "10만 유저"로 너무 늦음. "연 1회 + Paid CPU quota 80% 도달" 트리거 필요

4. **[Backend D-4] D1 FK PRAGMA 미들웨어 잘못된 안전감**
   - "요청마다 connection per request이라 safe" 주석은 D1 HTTP 게이트웨이 세션 보장 없음 (실제는 쿼리별 세션 가능)
   - 해결: `DB.batch([PRAGMA, query])` 패턴으로 같은 세션 보증

5. **[Quality QC-3] PaymentProvider Contract Test Suite 부재**

   - Phase 3에서 Polar/PortOne/토스가 각기 다르게 통과하는 경로가 열려 있음

6. **[Backend D-3] Dexie v1→v2 upgrade 콜백 부재**

7. **[DevOps D-C1/D-C2/D-C3] CI + 관찰가능성 + 환경 분리 전무**

## 반론 종합 (Devil's Advocate)

3개의 교차 확인된 "프로덕션 폭발 시나리오":

### 시나리오 1: 시험 직전 1주일 load 붕괴

손해평가사 시험 직전 1주일, 동시 접속 1,000명 × AI 튜터 peak. PBKDF2 600k + mathjs 160KB 번들 + D1 PRAGMA 왕복이 결합 → **Workers isolate cold start spike로 "시험 전날 접속 불가" 재앙**. 사용자 집단 환불 + SNS 바이럴.

### 시나리오 2: 2년차 교재 개정 시 constants 실수 소실

주니어 개발자가 "작은 오타 수정"으로 `UPDATE constants` 실행. D-1 지적대로 트리거 보호 없음 → Temporal 이력 전부 소실. 2028년 감사 시 "2026년에는 0.65였는가 0.60이었는가?" 대답 불가 → **기출 해설의 정답 근거 붕괴 → 사용자 집단 환불 소송**.

### 시나리오 3: Phase 3 Polar → PortOne 전환 시 인터페이스 재작성

Polar가 한국 결제수단 미지원으로 확인되어 PortOne 채택 → `createCheckoutSession`, `verifyWebhook`, `refund` 세 메서드 모두 **semantic gap** 발생. `providerOptions`에 꼼수 삽입. 결국 **MockProvider 테스트만 통과하고 실제 PG 연동은 어댑터 네이티브 직접 호출**. 인터페이스가 허공에 존재. 6개월 후 전면 리라이트.
