## Devil's Advocate 잔존 (비차단)

1. **dummy hash 통계적 구분**: DUMMY_HASH 가 all-zero 기반 → 실제 사용자 hash (무작위 분포) 대비 cache line/branch predictor 동작 미세 차이 가능. 수십만 샘플 통계 공격 가능성 잔존. Phase 1 Step 1-2 에서 실제 PBKDF2 산출물로 교체 권고.
2. **CF-Connecting-IP 'unknown' fallback**: 헤더 부재 시 모든 요청이 같은 키로 묶임. Production 은 CF 엣지 통과 시 항상 있음. Staging 자동화 테스트에서 false positive 가능.
3. **namespace_id 환경 분리**: 모든 환경이 `1001/1002` 공유 → staging 트래픽이 prod 카운터 소모 가능. 환경별 분리 (dev 1001/1002, staging 2001/2002, prod 3001/3002) 권고 — Step 1-2 에서 조정 가능.
