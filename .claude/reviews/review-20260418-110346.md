## CRITICAL 2건 — 양 에이전트 공통 지적

### C-1. `emit()` 최종 `JSON.stringify(record)` fallback 부재 → silent log drop

**위치:** `packages/shared/src/logger.ts:167`

**문제:**

- `merged` context 에 순환 참조(Hono `Context`, D1 PreparedStatement 래퍼), BigInt(D1 INTEGER PRIMARY KEY), Symbol 키 등이 포함되면 `JSON.stringify` throw
- 호출 스택이 상위로 전파 → 요청이 500으로 죽음
- 특히 error 경로(`log.error(...)`) 에서 터지면 **원래 사건 + 로깅 실패 둘 다 침묵**
- CLAUDE.md Hard Rule #3 "try-catch에서 데이터 조용히 삭제 금지" 정면 위반

**수정:** `emit` 전체 try-catch + fallback 최소 필드 JSON (`level`, `message`, `service`, `timestamp`, `_serializationError`).

### C-2. 중첩 객체/배열 PII 마스킹 부재 → PIPA 누수 구조적 취약

**위치:** `packages/shared/src/logger.ts:105-111`

**문제:** `maskContext` 는 top-level 1-depth 만 순회. 실무 패턴 대부분이 중첩:

```typescript
log.info('webhook', {
  buyer: { businessRegistrationNumber: '123-45-67890' }, // 미마스킹
  users: [{ password: 'hunter2' }], // 배열 내부 원본
  rawPayload: JSON.stringify(webhookBody), // 전체 JSON 원본
});
```

**수정:** 재귀 마스킹 + 깊이 제한(6) + 배열 순회 + 순환 참조 가드 + `RAW_DUMP_KEYS` 세트 (`rawPayload`, `body`, `headers`, `query`, `params`).
