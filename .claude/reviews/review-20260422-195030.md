## 🟠 Major 4건

### M-A1. `cache-policy.ts` 가 `/api/webhooks/*` 를 PRIVATE 목록에 포함하지 않음

- `cache-policy.ts:20-25` `PRIVATE_PATH_PREFIXES` 누락
- webhook 응답 `{ id: rowId, replayed }` 이 공용 캐시에 잡힐 경우 replay 로직 오동작 가능
- fallback `no-store` 덕에 실제 위험 낮으나 **보안 경계 선언 누락**
- 수정: `PRIVATE_PATH_PREFIXES` 에 `/api/webhooks/` 추가 (1줄)

### M-A2. `/refresh` 의 `revokeSession` + `createRefreshSession` 비원자 순서

- `routes.ts:468-485`
- revoke 성공 후 insert 실패 시 사용자 강제 로그아웃
- D1 batch 로 완전 atomic 은 crypto 연산 때문에 불가
- 해결안: **INSERT 먼저 → 성공 후 revoke** 순서 역전. 실패 시 중복 활성 세션만 남음 (보안 손실 없음, rotation 재시도로 복구 가능)

### M-A3. `verifyAndRotateRefreshSession` 함수 plan 명세 vs 구현 불일치

- `current.plan.md:93` 명세 있으나 `session.ts` 에 없음. rotation 로직이 `routes.ts:467-495` 에 inline
- 이미 TD-026 으로 등록됨 (기존 리뷰)

### M-A4. **CORS 완전 부재** — 브라우저 직접 호출 불가

- `index.ts` 전체 CORS 관련 코드 0건
- 현재 API-only 단계라 즉시 장애는 아니나 **Phase 2 프론트 개발 시작 시 블로커**
- staging 배포 후 진산님이 브라우저에서 테스트 시 preflight 전부 실패
- 수정: `hono/cors` 미들웨어 + `credentials: true` (쿠키 필수)
