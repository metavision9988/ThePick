### C-1. MockPaymentProvider 프로덕션 유출 경로 (3개 에이전트 동시 지적)

**위치:** `packages/payment/src/index.ts:2`, `packages/payment/src/providers/mock.ts`

**문제:**

- `index.ts`가 `MockPaymentProvider`를 barrel로 재노출 → 프로덕션 번들 tree-shaking 전 import 가능
- `mock.ts`에 `NODE_ENV/production` 가드 없음
- `signature !== 'mock-signature'` 하드코딩만으로 인증 통과

**공격 시나리오:** 운영자가 긴급 fallback을 `mock`으로 설정 → `POST /api/webhook/payment` + `X-Signature: mock-signature` + `{"userId":"victim","transactionId":"x","amountKRW":99000}` 한 번에 **임의 사용자 유료 승격**. PostHog 알람까지 수 시간 지연.

**수정:**

1. `index.ts`에서 `MockPaymentProvider` export 제거 (subpath `./mock`로만 노출)
2. `MockPaymentProvider` 생성자에 프로덕션 가드
3. `verifyWebhook`에 Zod 스키마 + 별도 에러 클래스(`WebhookSignatureError`, `WebhookPayloadError`)

### C-2. ADR-005 `timingSafeEqual` 미정의 (Pass 3)

**위치:** `docs/adr/ADR-005-authentication-pbkdf2-sha256.md:70`

**문제:** 호출만 있고 함수 정의 없음. Workers는 `crypto.subtle.timingSafeEqual` 미지원. Phase 1 구현자가 `a.every((b,i)=>b===expected[i])` 식 조기반환 비교 시 바이트별 타이밍 누출.

**수정:** ADR-005에 reference 구현 추가:

```typescript
function timingSafeEqual(a: Uint8Array, b: Uint8Array): boolean {
  if (a.length !== b.length) return false;
  let r = 0;
  for (let i = 0; i < a.length; i++) r |= a[i] ^ b[i];
  return r === 0;
}
```
