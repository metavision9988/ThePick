## Devil's Advocate 권고 → 반영

**시나리오:** `wrangler deploy` (--env 생략) 실행 시 default env의 placeholder UUID로 Cloudflare API 호출 시도 → "database not found" 에러 위험

**대응 완료:** `apps/api/package.json`의 `deploy` 스크립트를 **명시적 에러 메시지**로 변경:

```json
"deploy": "echo 'ERROR: Use deploy:staging or deploy:production (default env deploy is prohibited)' && exit 1"
```

→ default env로 실수 배포 원천 차단. 개발자는 `deploy:staging` / `deploy:production` 중 반드시 명시적 선택.
